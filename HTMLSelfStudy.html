Introduction :

Une page html commence toujours avec la balise <!DOCTYPE html>.
Cela permet de préciser que le type de cette page est du HTML.

HTML signifie HyperText Markup Language.

Le "X" de XHTML viens de XML : eXtensible Markup Language.

La première page d'un site doit se nommé "index.html".
En effet c'est sur cette page là sur laquelle tomberont les internautes si il tape l'adresse de votre site sans préciser le nom d'un fichier.
Si il n'existe pas , ils tomberont sur une liste de fichier comme ci-dessous:

Quel que soit le nombre d'espaces ou de retour à la ligne (autre qu'avec br) dans une balise, le navigateur n'affichera qu'un seul espace.
On peut donc pour plus de lisibilité, en mettre autant que l'on veut dans le code html.

Balises HTML :

L'utilisation de balises présente plusieurs intérêt :

  • Le code est plus clair
  • La future maintenance du site sera facilitée
  • Les moteurs de recherche indexeront mieux les pages

Voici plusieurs balises essentielles :

  • <html> : Définit un document html.
  • <head> : Définit l’en-tête d’un document.
  • <title> : Définit le titre d’un document.
  • <meta> : Définit des informations relatives au document.
  • <link> : Définit les relations entre les documents.
  • <body> : Définit le corps de la page.
  • <p> : Définit un paragraphe.
  • <h1> à <h6> : Définit un titre par degré importance de 1 à 6.
  • <a> : Définit un lien.
  • <ul> (unordered list): Définit une liste non-ordonné.
  • <ol> (ordered list): Définit une liste ordonné.
  • <li> : Élément d’une liste.
  • <code> : Mise en forme d’un texte en code informatique.
  • <pre> : Texte pré-formaté.
  • <div> : Définit un calque ou une section.
  • <nav> : Définit un groupe de liens de navigation.
  • <article> : Définit un article.
  • <section> : Définit une section.
  • <aside> : Définit une partie latérale au contenu.
  • <header> : Définit le haut d’une section ou d’une page.
  • <footer> : Définit le bas d’un section ou d’une page.
  • <br> : Saut de ligne.
  • <nav> : Définit un groupe de liens de navigation.
  • <img> : Définit une image.
  • <table> : Définit un tableau.
    • <thead> : Définit le haut d’un tableau.
    • <tbody> : Définit le corps d’un tableau.
    • <tfoot> : Définit le bas d’un tableau.
    • <tr> : Définit une ligne de tableau.
    • <td> : Définit une cellule d’un tableau.
    • <th> : Définit une cellule d’en-tête d’un tableau (à apprendre).
  • <dl> : Définit une liste de définition
  • <dt> : Définit le titre d'une définition
  • <dd> : Définit le texte d'une définition
  • <strong> (stronger emphasis): Permet d’indiquer à un moteur de recherche qu’un contenu est particulièrement important.
  • <em> (emphasis): Ressemble à strong mais moins important.
  • <mark> : Sert à faire ressortir du contenu.
  • <div> (division): Sert à englober un bloc de contenu.
  • <span> : Sert à englober du contenu.
  • <base> : Définit un dossier par défaut pour l'ensemble des liens.
  • <sup> : Met un caractères en exposant.
  • <sub> : Met un caractères en indice.
  • <small> : Réduit la taille d'un texte.
  • <blockquote> : Permet de faire une citation.
  • <hr> (Horizontal rule): Trace un trait de séparation sur toute la largeur.

Commentaires et caractères spéciaux :

On peut écrire un commentaire dans le code HTML avec :
<!-- Mon super commentaire -->
Attention cependant car tout le monde peut voir un code source d'une page HTML donc éviter les données sensibles.

Un caractères spécial commence toujours pas "&" et finis toujours par un ";".

Pour forcer l'affichage de plusieurs espaces successifs sur une ligne, il faut utiliser le caractères spéciale "&nbsp;"
Ce caractères correspond au "non-breaking space". Il a pour propriété d'empecher la coupure en fin de ligne s'il est placé entre deux mots.
Pratique notamment pour les prix "10 €" ou les heures "10 h".

Quelques caractères spéciaux communs :

  • &ccedil; = ç
  • &euro; = €

Attributs :

Attributs universel

L'attribut "title" donne des informations supplémentaires sur un élément.
<p title="exempleTitle">

L'attribut "id" permet de donner un id à une balise.
<section id="monSuperID">
Un ID sert notamment pour les ancres. Tout ID est évidemment unique (sur la même page).

L'attribut universel "class" indique une liste de classes associées à l'élément courant. Les classes permettent de manipuler les éléments,
via CSS ou JavaScript en utilisant les sélecteurs de classe ou des fonctions telles que document.getElementsByClassName.
<p class="maSuperClasse">Mon super texte<p>

Attributs de la balise <img>

L'attribut "src" définit la source de l'image à afficher.
<img src="lienDeMonImage"/>

L'attribut "alt" sur une balise <img> affiche un texte alternatif si l'image n'est pas chargée.
<img src="lienDeMonImage" alt="Mon texte alternatif"/>

Attributs de la balise <meta>

L'attribut "charset" spécifie l'encodage des caractères du document HTML.
<meta charset="utf-8">

Attributs de la balise <ol>, <ul> et <li>

L'attribut start="..." sert à indiquer le numéro de départ.

L'attribut "reverse" sert à avoir une numérotation décroissante.

L'attribut type="..." permet de changer le style de numérotation. (1,2,3 / A,B,C...).

L'attribut "value" d'une balise li permet de spécifier un numéro pour une ligne donné.

Attributs de la balise <a>

L'attribut "href" (HyperText reference) permet de définir le lien d'une balise <a>.
<a href="monSuperLien">Le texte du lien</a>
On peut également faire référence à une autre page HTML ou bien un élément de la page (Ancre) dans href afin de faciliter la navigation.
Noter qu'il faut utiliser des "/" et des non des "\" dans les chemins de fichiers.

Pour créer une ancre il suffit de donner l'id de la cible avec un # devant dans href.
Il est également possible de combiner un chemin de fichier avec une ancre :
<a href="monSuperLien#maSuperAncre">Le texte du lien</a>

Pour créer un lien de téléchargment, il suffit de donner le lien du fichier à télécharger dans href.
Celui-ci étant différent d'un fichier.htlm, il sera téléchargé.

Pour ouvrire un lien dans une nouvelle page, on utiliser l'attribut "target" avec la valeur "blank".
<a href="monSuperLien" target="_blank">mon super texte</a>
target sert aussi au <iframe> (déconseillées).

Pour qu'un lien envoie un mail à une certaine adresse, on ajoute "mailto:" puis le mail dans l'attribut "href".
<a href ="mailto:exemple@mail.com">mon super texte</a>
Le logiciel de messagerie par défaut sur l'ordinateur de l'utilisateur sera utilisé.
Il est possible d'ajouter un sujet et d'autre destinataire avec le séparateur "?" après l'adresse mail puis avec le séparateur "&" avant chaque nouveau paramètre.
<a href ="mailto:exemple@mail.com?subject=Mon message&CC=titi@gmail.com">mon super texte</a>

Pour qu'un lien amène à un numéro de téléphone, on ajoute "tel:" puis le numéro dans l'attribut "href".
<a href ="tel:0123456789">mon super texte</a>

L'attribut title permet d'affichier automatiquement du texte dans une bulle au passage de la souris.

Autres

L'attribut "lang" précise la langue utilisée pour le contenu de la page, il est placé sur la balise <html>.
<html lang="fr">
Une langue bien indiquée sera utile aux synthèses vocales (pour adopter le bon accent).

L'attribut "cite" de la balise <blockquote> permet de donner la référence de la citation.

Tips et bonnes pratiques :

On ne met qu'une balise <h1> par document HTML.

Ne jamais mettre de contenu dans une balise <hX> juste pour l'avoir en grand et gras.
C'est le CSS qu'il faut utiliser.

Dans une liste de définition, <dt> se trouve toujours avant <dd>.

Les balise <strong> <em> et <mark> servent au référencement de site. (Moins utilisés de nos jours).

Un page web peut s’afficher différemment selon les navigateurs (moins maintenant) et sur de vieux navigateurs certaines fonctionnalités ne marcheront pas.
Il faut donc tester son code sur plusieurs navigateurs.

Il faut toujours avoir un code HTML et CSS valide car critère de référencement.
On peut alors utiliser des validateurs de code :

  • Validateur HTML
  • Validateur CSS

Un nom de fichier dois éviter de contenir des espaces et caractères spéciaux.

Les balises s'écrivent toujours en minuscules et doivent être fermées.

Un paragraphe <p> ne contient jamais d'autre paragraphes.

Une balise <small> doit correspondre à un contenu accessoire et indépendant du contenu principal et ne dois pas servir à de la mise en forme.

Les balises <dt> et <dd> servent aussi pour des dialogues. ("dt" pour dialog talker et "dd" pour dialog discourse)

Tableaux HTML :

L'utilisation de tableaux est réserver à la présentation de données et non à de la mise en forme.
Délimité par la balise <table>, initialement les bordures sont invisibles. On peut changer l'épaisseur avec l'attribut "border" (mais privilégier le CSS).

La table se construit ligne par ligne à l'aide de la balise <tr> (table row).
Chaque lignes contient une cellule <td> (table data). Lorsqu'il s'agit d'une cellule de titre on utiliser <th> (table header).

Les balises <thead> et <tfoot> servent pour l'en-tête et le pied de tableau.
Elles sont écrites avant le corps du tableau (qui doit alors être dans un <tbody>).

La balise <caption> est facultative et permet de définir un titre associé au tableau.

Pour mettre une mise en forme à des colonnes précises, on peut utiliser des balises <col> réunit dans une balise <colgroup>

Les attributs "colspan" et "rowspan" d'une balise <td> servent respectivement à fusionner les cellules de plusieurs colonnes et plusieurs lignes.
